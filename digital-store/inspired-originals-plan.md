# Inspired-Originals Build Plan (Practice Forward Derivative-Safe)

## Objective
Create commercially safe, original digital products that are **inspired by outcomes/workflows** from Practice Forward, but do **not** copy protected expression.

## Hard Rules (Non-Negotiable)
1. Do not copy/paste any sentence, paragraph, table labels, slide titles, or template language from source files.
2. Do not reuse original file names, module names, branded terms, or layout structures 1:1.
3. Rebuild all calculators from scratch (new cell structure, labels, formulas layout, examples).
4. Rewrite all frameworks in your own method language (new acronyms, new sequencing, new visuals).
5. Use only generic accounting/tax concepts and your firm’s own positioning.
6. Add clear attribution policy internally: "Inspired by industry best practices; original RHW materials."

## Build Framework
For each product, create 5 assets:
1. **Method Doc** (the proprietary framework)
2. **Client-Facing Workbook**
3. **Advisor Script/Runbook**
4. **Calculator(s)**
5. **Sales Page + Fulfillment Email**

## Naming Conventions (Original)
- Prefix all internal versions with `RHW-`.
- Example: `RHW-Entity-Decision-Playbook-v1.xlsx`
- Avoid terms used in source package names.

## QA Checklist (Derivative Safety)
- [ ] No identical headings from source
- [ ] No identical sentence strings (spot-check with search)
- [ ] New visual structure and sequence
- [ ] New examples/case data
- [ ] RHW branding and voice throughout
